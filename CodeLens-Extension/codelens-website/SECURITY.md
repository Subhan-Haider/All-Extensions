## Reporting a Vulnerability

We take the security of CodeLens and its users very seriously. If you have found a security vulnerability in the CodeLens Website or the Extension, we appreciate your help in disclosing it to us responsibly.

### How to report

Please do **DO NOT** open a public issue on GitHub for sensitive security vulnerabilities.

Instead, please report vulnerabilities by:

1.  **Email**: Send a detailed report to [security@codelens.sec](mailto:security@codelens.sec) (Note: Replace with actual contact if available, otherwise use GitHub Security Advisories).
2.  **GitHub Security Advisories**: If enabled on the repository, please use the "Report a vulnerability" tab to open a private advisory.

### what to include

In your report, please include:

*   The version of CodeLens affected.
*   A detailed description of the vulnerability.
*   Steps to reproduce the issue (POC code or screenshots are highly appreciated).
*   The potential impact of the vulnerability.

### Response Timeline

*   We will acknowledge receipt of your report within **48 hours**.
*   We'll provide a status update every **5 business days**.
*   We aim to fix critical vulnerabilities within **14 days**.

## Security Best Practices for Users

*   **Official Sources**: Only install the CodeLens extension from the official Chrome Web Store, Edge Add-ons Store, or this verified GitHub repository.
*   **Updates**: Keep your extension and browser updated to the latest version.
*   **Permissions**: CodeLens is designed to require minimal permissions. Be wary of any version asking for excessive access.
